# V21 11国最终定稿版融合执行策划案验收器与人工审核清单

适用对象：11国/多项目 V21 最终定稿版融合执行策划案 HTML。  
依据：`v21-final-exec-standard.md` 与 `v21-11country-audit.md`。  
结论原则：**自动检查只能判定结构与显性风险；最终命名为 final 前必须通过人工硬审。任一硬失败项存在，文件只能进入 `internal-review` / `draft` / `structure-preview` / `fallback-test`。**

---

## 0. 验收状态定义

| 状态 | 含义 | 可否客户交付 | 可否使用 final 命名 |
|---|---|---:|---:|
| `V21_FINAL_READY` | 自动检查通过 + 人工硬审全通过 | 是 | 是 |
| `V21_INTERNAL_REVIEW` | 结构基本存在，但存在数据/税务/法案/佣金/来源等硬缺口 | 否 | 否 |
| `V21_STRUCTURE_PREVIEW` | 仅完成融合框架、章节、模块入口，未证明内容完整 | 否 | 否 |
| `V21_FALLBACK_TEST` | 任一选中项目仍使用 V20Plus/fallback/旧源 | 否 | 否 |
| `V21_FAILED` | 缺核心结构、坏图、乱码、404、严重不可读或明显削薄 | 否 | 否 |

## 2. 人工硬审项

> 以下为 final 前人工必须逐项签字的硬标准。任一项不通过，不得命名 final。

### 2.1 前置源页硬审

- [ ] 每个 `国家 × 项目 × 强相关子路径` 已独立生成合格 V21 单项目页。
- [ ] 每个单项目页独立包含完整 15 章。
- [ ] 每个单项目页含第12章费用/材料/三方费用/三方材料。
- [ ] 每个单项目页含第14章财税执行策划案全文。
- [ ] 每个单项目页含第15章法案条款/官方资源/执行动作。
- [ ] 每个单项目页含真实佣金/成本页。
- [ ] 无任何 V20Plus-fallback 项目参与最终融合。

硬失败：任一项目未先单独通过 V21。

### 2.2 融合逻辑硬审

- [ ] 融合是“15章横向归并”，不是多国摘要。
- [ ] 每章按国家/项目依次排列。
- [ ] 只去重完全重复通用声明/重复图，不删除国家差异。
- [ ] 没有删除教育、福利、生活费用、税务、跨境、流程、法案、佣金等内容。
- [ ] 最终厚度接近所有选中单项目完整页之和。

硬失败：用短表格替代单项目全文，或把 11国页做成摘要型 briefing。

### 2.3 真实数据硬审

- [ ] 投资额、官方费、律师费、顾问费、翻译公证、认证、体检、保险、银行、税务师、审计、维护费均有当前具体金额/公式。
- [ ] 服务费、底价、佣金、结佣周期、结佣批次、对接人真实完整。
- [ ] 每个关键数据有来源类型、来源日期或版本。
- [ ] 确实无法固定报价的非核心项，已写明未知原因、核验方式、影响范围。
- [ ] 没有用“按官方/以官方为准/递交前复核”替代当前应有核心数据。

硬失败：核心费用、投资门槛、服务费、底价、佣金、三方费大面积待确认。

### 2.4 财税有效性硬审

人工必须逐项目确认第6章和第14章回答了：

- [ ] 该项目具体解决什么税务问题。
- [ ] 如何判定/切换税务居民。
- [ ] 境内企业利润、分红、股权转让、服务费、IP费、投资收益如何处理。
- [ ] 资金从中国到境外、从境外到家庭账户/公司/投资主体的合规路径。
- [ ] 如何避免双重征税、降低预提税/资本利得/所得税/遗产赠与税等税负。
- [ ] CRS、CFC、FATCA、FBAR、PFIC、转让定价等是否按适用国家正确说明。
- [ ] 37号文、ODI、FDI、返程投资、外汇和银行 KYC/KYB 触发点是否清楚。
- [ ] 图形与文字一致，不是通用 SPV 图套壳。

硬失败：财税架构是通用模板，未证明项目化税务效果。

### 2.5 法案条款硬审

每个项目第15章必须达到：

- [ ] 正式法案/政策/官方资源名称准确。
- [ ] 条款号、章节号、政策手册段落或官方页面具体规则明确。
- [ ] 每条说明适用条件、风险与执行动作。
- [ ] 能指导客户、律师、会计师、项目方下一步工作。
- [ ] 官方链接或文件来源可靠，且版本日期清楚。

硬失败：只列官网类别或资料类别，不解释条款适用。

### 2.6 费用与材料硬审

- [ ] 每个项目费用拆到付款方、收款方、节点、币种、是否可退、来源。
- [ ] 三方费用有独立报价/公式和责任边界。
- [ ] 投资款、服务费、生活费、税费、维护费不混写。

- [ ] 每项材料有责任人、用途、提交节点、审核风险。

硬失败：只有泛化材料类别或总预算，无可执行明细。

### 2.7 佣金页硬审

- [ ] 密码 `888888` 可进入。
- [ ] 只列选中项目及强相关子路径。
- [ ] 国家、项目、服务费、底价、佣金、结佣周期、结佣批次、对接人、状态逐行完整。
- [ ] 无“详见源页”“待确认”“根据实际情况”替代核心商业数据。

硬失败：佣金页大量待确认或只有引用。

### 2.8 视觉与可读性硬审

- [ ] 手机端可读，宽表/宽图可横向滚动。
- [ ] SVG/图片没有坏图、乱码、裁切、遮挡。
- [ ] 深底白字、浅底深字，业务图文字清晰。
- [ ] 没有全局 SVG 文字颜色覆盖造成不可读。
- [ ] 页面整体达到商业应用专业级美感：深海军蓝、克制金色、白底卡片、统一表格、统一圆角/描边/阴影。
- [ ] 没有廉价渐变块、杂乱阴影、emoji 堆叠、旧PPT模板感。

硬失败：核心图表或表格在手机端无法读清。

## 3. final 文件命名条件

### 3.1 允许使用 final 命名的条件

只有同时满足以下条件，文件名才允许包含 `final`、`v21-final`、`commercial-final` 或作为客户最终审核版发布：

1. 自动检查全部通过，且无 blocker / hard-fail。
2. 11国全部最小单元源页均为 V21-professional / V21_OK 或更高状态。
3. 无任何 fallback、V20Plus、draft、preview、test 源页参与融合。
4. 完整单项目模块可展开或单独打开。
5. 15章完整，且每章按国家/项目横向归并。
6. 第12章费用、材料、三方费用、三方材料、投资款、生活费完整。
7. 第14章财税执行策划案可证明项目化税务解决方案。
8. 第15章法案条款达到条款/章节/政策段落颗粒度。
9. 佣金页密码为 `888888`，且真实结构化数据完整。
10. 0 坏图、0 乱码、0 SVG不可读、0 核心表格手机不可读。
11. 人工硬审清单全部打勾，并留存审核记录。

推荐 final 命名格式：

```text
fusion-11country-v21-final-review.html
fusion-11country-v21-final-commercial.html
fusion-<组合标识>-v21-final-<YYYYMMDD>.html
```

### 3.2 禁止使用 final 命名的情形

出现任一情况，必须改用非 final 命名：

```text
fusion-11country-v21-internal-review.html
fusion-11country-v21-structure-preview.html
fusion-11country-v21-fallback-test.html
fusion-11country-v21-draft.html
```

禁止命名为 final 的情形包括：

- 任一项目仍为 V20Plus/fallback。
- 任一项目源页缺失、404、打不开。
- 核心费用、服务费、底价、佣金、投资门槛、三方费待确认。
- 财税架构为通用模板，没有项目化税务效果。
- 法案附件只是官网/机构类别，没有条款级解释。
- 佣金页只有“详见源页”或大量待确认。
- 页面只有结构和数量达标，但人工硬审未通过。

- 15章标题齐全但章节正文为短摘要/短表格。
- 第12章只有总预算，没有费用、三方费用、材料、三方材料分表。
- 第14章只有通用 SPV/家办图，没有税务居民、资金流、预提税、CFC/CRS/FATCA/FBAR 等项目化说明。
- 第15章只有机构官网名称，没有条款适用、风险、执行动作。

### 4.4 技术与视觉降级结构

- 全局 CSS：`svg text { fill: ... !important }` 或类似强制覆盖 SVG 字色。
- 大量 `undefined`、`NaN`、`[object Object]`、模板变量外露。
- 图片 404、SVG 空白、iframe 加载失败。
- 深色框深色字、浅色框浅色字。
- 宽表在手机端被裁切且不可横向滚动。

## 7. 一票否决规则

出现任一项，直接不得 final：

1. 任一项目仍为 V20Plus/fallback/旧源。
2. 任一项目源页缺失或 404。
3. 缺完整单项目模块或只能看到摘要。
4. 缺 15 章任一章，尤其第12/14/15章。
5. 核心费用、服务费、底价、佣金、投资门槛、三方费大面积待确认。
6. 财税架构为通用模板，不能说明项目化税务解决。
7. 法案条款只有官网类别，无条款/章节/政策段落。
8. 佣金页未设密码 `888888`，或大量待确认/详见源页。
9. 坏图、乱码、SVG文字不可读、手机端核心表格不可读。
10. 人工硬审未完成或未通过。

## 8. 建议工作流

1. 先跑自动验收器，输出 `V21_INTERNAL_REVIEW` 或 `V21_FINAL_CANDIDATE`。
2. 若命中 fallback/待确认/通用财税/类别级法规，立即停止 final 命名。
3. 回到单项目页逐个修复，直至所有最小单元 V21_OK。
4. 重新融合，不在失败融合页上继续堆内容。
5. 再跑自动验收器。
6. 自动无硬失败后，执行人工硬审清单。
7. 人工硬审全部通过后，才允许生成 final 文件名与客户审核链接。
