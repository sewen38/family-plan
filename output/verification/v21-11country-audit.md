# V21 11国随机组合最终审核页差距审计报告

审计对象：`family-plan-pages/fusion-random-11country-v21-final-review.html`  
审计标准：`family-plan-full` Skill 内 V21 最终商业级执行策划案标准、`v21-final-exec-standard.md`、多国多项目融合标准、V21 人工审核新增硬标准。  
审计结论：**未达 V21 最终人工审核标准，不能作为最终版交付。**

## 一、总体判定

页面在形式层面已经具备：Hero、数据完整性检查表、完整单项目模块嵌入区、15章归并正文、费用/材料/三方费用/三方材料关键词、财税章节、法案章节、佣金页等结构。

但按 V21 最终人工审核硬标准，核心问题不是“有没有章节”，而是：

1. **部分项目仍明确使用 V20Plus fallback，并套 V21 融合包装**，不满足“每个选中项目必须先有合格 V21-professional 单项目完整页”的前置条件。
2. **真实数据不足**：大量核心费用、政府费、服务费、底价、佣金、基金发行费、管理费、尽调费、三方费仍写“待确认 / 以官方规则为准 / 按官方费用表逐项核算 / 版本日期需复核”。这可用于内部草稿，不能用于 V21 最终交付。
3. **财税架构对税务问题的解决不够具体**：部分国家只给“中国主体—境外SPV—区域公司—信托/基金会—家庭账户”等通用模板，未按国家/项目说明如何降低税负、避免双重征税、降低预提税、处理 CFC/CRS/FATCA/FBAR、利润归集、税务居民切换、37号文/ODI/FDI 与资金回流。
4. **法案条款颗粒度不均衡**：澳大利亚 482、部分土耳其内容较接近要求；但日本、美国、多米尼克、格鲁吉亚、马耳他等多处仍是“官方移民局/内政部、税局/财政部、项目法案/条例/政策手册”这种类别级占位，没有逐条列出正式法案名称、条款/章节/政策段落、适用条件、风险、执行动作。
5. **佣金页不合格**：多国总佣金页写“详见对应单项目源页佣金/成本区”“通常3个月或按项目进度”“根据实际情况”；多个单项目底价、佣金、对接人、结佣批次为待确认。V21 要求佣金页结构化展示选中项目的服务费、底价、佣金、结佣周期、结佣批次、对接人、状态，不得用泛化引用替代。
6. **所谓“完整单项目模块嵌入区”更多是源页链接/展开入口，不等于所有项目已通过 V21 单项目验证**。尤其 fallback 项目不能因被嵌入/链接就视为 V21 合格。

本页 HTML 规模约 823KB，提取正文约 17.4万字、表格 499 个、SVG 28 个、图片 28 个。数量指标看似足够，但 V21 新硬标准明确：**结构、表格、SVG 达标不代表通过；缺真实数据、缺有效财税避税架构、法案条款不详细，一律未达标。**

## 二、失败原因分类

### A. 前置单项目质量失败：fallback 冒充 V21 最终源

页面自身“数据完整性检查表”明确显示以下项目为 `V20Plus定稿源 / V21融合包装`：

- 美国 · EB-1A 杰出人才绿卡：`final-single/us-eb1a-v20plus-final.html`
- 多米尼克 · CBI 捐款入籍：`final-single/dominica-cbi-v20plus-final.html`
- 格鲁吉亚 · 投资居留路径：`final-single/georgia-investment-residence-v20plus-final.html`
- 马耳他 · MPRP 永居路径：`final-single/malta-mprp-v20plus-final.html`

V21 规则明确：fallback 只能作为旧参考，必须先升级为合格 V21-professional 单项目页，再参与最终商业级融合。当前页面把 fallback 项目放进 11 国融合，是根本失败项。

### B. 数据真实性失败：核心数据大量待确认或泛化

典型问题：

- 土耳其基金入籍费用表中：基金认购/发行费、基金管理费/托管费、政府申请费、基金尽调费、医疗保险/居留配套、服务费、E-2 二阶段评估费等多项为“待确认”。
- 日本费用表中：行政书士、翻译公证、服务费/底价/佣金等仍待确认或区间过宽。
- 香港、美国、多米尼克、格鲁吉亚、马耳他佣金/成本均存在大面积待确认。
- 多国总佣金页没有真实底价、真实佣金、真实结佣批次、真实对接人，只写“详见对应单项目源页佣金/成本区”。
- 部分费用写“按官方费用表逐项核算”“递交前复核版本”，属于流程提醒，不是已核验数据。

V21 要求费用、投资额、官方费、律师/顾问费、生活费、税率、门槛、时间、预算、结佣信息必须真实且详细；确实未知时要写未知原因、核验来源、影响范围，但不能用未知项替代核心交付数据。当前不满足。

### C. 财税架构有效性失败：通用架构多，项目化税务解决不足

页面反复出现类似表述：

- “中国主体—境外SPV—区域公司—信托/基金会—家庭账户”
- “嵌入完整财税执行策划案框架：ODI、37号文、红筹、SPV、CRS、CFC、转让定价”
- “按客户企业和家庭信息替换”
- “用于本项目落地时按某项目替换参数”

这些表达说明财税内容仍是通用框架/参数替换思路，而不是针对每个国家/项目的税务问题解决方案。V21 要求第6章/第14章必须解释：

- 税务居民如何判定与切换；
- 境内企业利润、分红、股权转让、服务费、知识产权费、投资收益如何归集；
- 哪个主体持有资产，哪条资金流合规出境；
- 如何避免双重征税、降低预提税、处理 CFC/CRS/FATCA/FBAR；
- 何时需要 37号文、ODI、FDI、返程投资、转让定价文档；
- 对客户企业和家庭税负有什么实际优化结果。

当前很多国家没有把上述内容落到项目和法域，尤其 fallback 项目基本是财税模板套壳，未能证明“解决税务问题”。

### D. 法案条款失败：法规附件存在类别级占位

合格示例：澳大利亚 482 有较具体条款，如 Migration Act 1958 s140E、s245AR/s245AS，Migration Regulations 1994 reg 2.59、reg 2.72，Subclass 482、Subclass 186、Fair Work Act 等，并说明客户动作和风险。

不合格示例：日本、美国、多米尼克、格鲁吉亚、马耳他等章节出现统一模板：

- 官方移民局/内政部：项目资格、表格、审理流程、费用表；
- 税局/财政部：税务居民、个人税、公司税、资本利得、申报期限；
- 项目法案/条例/政策手册：投资门槛、持有期、续签/入籍、尽调标准；
- 学校/医保/福利官方资料：教育、医疗、福利资格。

这不是 V21 所要求的“法案/政策名称 + 条款/章节/政策段落 + 适用项目 + 适用条件 + 约束/风险 + 执行动作”。类别级来源不能通过人工审核。

### E. 费用清单 / 材料清单 / 三方费用规则未完全满足

页面有大量表格，但存在以下硬伤：

- 某些项目第12章只是通用模块表，未逐项展开官方费、律师费、翻译公证、体检、认证、投资款、维护费、退出费、年度税务/会计费、家庭生活费。
- 三方费用有标题，但对律师、税务师、审计师、银行、基金管理人、房产方、项目方、保险、学校等第三方角色的独立报价、付款节点、责任边界、发票/合同要求不完整。
- 三方材料有标题，但部分国家没有把“客户侧 / 项目方侧 / 律师顾问侧 / 税务师侧 / 银行侧 / 政府表格侧”逐项拆开。
- Fallback 项目的材料/费用大多是模板化“身份、家庭、资产、税单、企业、学历、无犯罪、体检”层级，不足以指导执行。

### F. 商业终审表达失败：页面自己暴露“待确认 / fallback / 版本待复核”

最终审核页不能出现大量“待确认”“版本日期需复核”“V20Plus定稿源 / V21融合包装”“用于落地时替换参数”等内部草稿语。它们说明页面仍处于结构拼装或初审状态，而非最终商业级定稿。

## 三、每个国家/项目不达标点

> 说明：以下按当前 11 个项目逐项列出。若同一国家有两个项目，按项目拆分。

### 1. 新加坡 · EP / EP-PIC 自雇投资公司

当前优点：使用 V21 商业美化单项目源；费用、材料、EP/PIC 逻辑相对完整；COMPASS、MOM、ACRA、IRAS、公司实质等方向基本正确。

不达标点：

- 客户身份、随行成员、税务身份仍为待确认。若作为随机测试页可以保留默认信息，但最终客户交付必须替换为真实客户信息。
- 财税部分需要进一步把 EP 雇主担保与 EP-PIC 自雇公司分开：工资薪金税、公司所得税、股东分红、管理费/服务费、境内企业与新加坡公司的关联交易、转让定价、CRS、CFC、37号文/ODI 触发点要逐项说明。
- 佣金页中“财税定制化执行策划案”底价/佣金/对接人仍待确认，属于商业内部数据缺口。
- 三方费用应明确新加坡公司秘书、会计师、税务师、审计、银行开户、办公室、雇佣/薪资服务、MOM 申请、ICA 后续 PR 材料的独立费用与付款节点。

整改要求：保留 V21 源页，但补齐客户真实字段、税务居民判断、EP/PIC 两条税务解决逻辑、三方费用明细、佣金底价/批次。

### 2. 香港 · ASMTP 专才 / 自雇公司路径

当前优点：使用 V21 专业最终单项目源；香港雇主真实性、岗位必要性、自有公司真实经营、7年通常居住等方向正确。

不达标点：

- 客户简历、学历、行业、家庭成员仍待确认。
- 香港佣金页中 ASMTP 雇主担保、自有公司/自雇专才的服务费、底价、佣金、结佣周期、结佣批次、对接人全部待确认。
- 法案条款应写清 ASMTP 具体官方政策页面、ID 990A/ID 990B 表格要求、输入内地人才计划适用条件、受养人政策、7年通常居住/永久性居民依据、香港薪俸税/利得税/地域来源原则条款或 IRD 指引，而不能只泛称官网。
- 财税架构需具体解决：香港公司与内地企业业务关系、服务费/顾问费/分红的税务性质、利得税地域来源、审计与商业实质、CRS、内地个人税居民和反避税风险。

整改要求：补齐香港法律依据和 IRD 指引颗粒度；把“自有公司专才”的经营证据、薪酬、审计、税务、续签、7年通常居住做成可执行材料链；补齐佣金页。

### 3. 土耳其 · 基金投资入籍 + 美国 E-2 跳板

当前优点：已有基金入籍与 E-2 二阶段联动；有费用、材料、E-2 衔接和土耳其/美国法律资源方向。

不达标点：

- 核心费用大量待确认：基金认购/发行费、管理费/托管费、政府申请费、基金尽调费、医疗保险/居留配套、服务费、E-2 二阶段评估费等。
- 基金投资认购款写“建议 USD 500,000 级别预审，最终以官方规则和基金文件为准”，需要明确现行土耳其投资入籍基金路径的法定门槛、合格基金类型、持有期、监管/托管证明、退出限制、收益税务处理和官方来源。
- 财税架构需说明土耳其税务居民触发条件、基金收益税务、美国 E-2 后是否触发美国税务居民、E-2 公司利润/工资/分红税务、FATCA/FBAR 风险，而不只是“基金 + E-2 跳板”。
- 法案条款中土耳其国籍法/实施条例和美国 9 FAM 有方向，但需补足具体条款号、官方链接、适用条件、客户动作、拒绝/补料风险。
- 佣金服务费、底价、对接人仍待确认。

整改要求：先核验土耳其基金 CBI 当前官方规则和基金文件样例；把基金方/律师/银行/税务师的费用与材料拆开；E-2 作为二阶段必须单独列法案条款、预算和税务风险。

### 4. 土耳其 · 房产投资入籍

当前优点：房产 USD 400,000 门槛、Tapu、估值、三年限制、交易税、公证、律师等方向较具体。

不达标点：

- 部分费用仍以“预估 / 实际以当地规则 / 按官方更新”为主，需标注数据来源日期和官方/市场报价来源。
- 专业服务费待确认；对接人待确认。
- 房产税务架构需更具体：购置税费、持有期间租金税、出售资本利得、遗产/赠与、土耳其税务居民触发、中国税务居民对境外房产收益申报、CRS 信息交换。
- 法案条款需从“Turkish Citizenship Law No.5901 / Implementation Regulation / TKGM / NVI”进一步落到具体条款、实施条例条款、房产持有三年限制规则、估值报告规则、家庭成员适用范围。
- 三方材料要明确卖方/开发商/律师/估值机构/Tapu/银行/翻译公证/税务师各自提供什么文件。

整改要求：保留现有框架，补法案条款号、官方链接、税务居民与房产收益税务、专业服务与佣金真实数据。

### 5. 澳大利亚 · 482 SID 技能需求签证

当前优点：V21 商业图形修正版源；法案条款相对最接近标准，列出 Migration Act、Migration Regulations、Subclass 482/186、CSIT/SSIT、Fair Work、SAF 等。

不达标点：

- 186 TRT 后续永居的服务费、底价、佣金仍待确认。
- 雇主端三方费用应进一步拆出 MARA/移民律师、雇主合规审计、工资/Super/PAYG、保险、英语考试、职业评估（如适用）、体检/无犯罪、翻译认证等费用来源。
- 财税架构需要更明确说明澳洲税务居民判定、临时居民/PR 阶段全球收入、Super、PAYG、家庭资产、境内公司分红、双边税收协定、CGT、FBT 等如何处理。
- 若客户是中国企业主，还需明确是否通过澳洲雇主真实雇佣，避免买职位、工资回流、SAF 转嫁等税务/劳动法/移民欺诈风险。

整改要求：保留澳洲法案条款颗粒度作为模板，补三方费用真实数据、186 TRT 后续商业条款、澳洲税务居民和中澳双重征税处理。

### 6. 日本 · 高度人才 / 经营管理签证

当前优点：已有 HSP/经营管理扩写、费用表、法规附件方向。

不达标点：

- 服务费、底价、佣金、结佣周期、结佣批次、对接人均待确认。
- 行政书士、翻译公证等费用仍“约 / 待确认”，缺报价来源和区间依据。
- 法律法规附件仍不足：需要列明出入国管理及难民认定法、入管法施行规则、高度专业职积分制告示/积分表、经营管理在留资格审查要领、永住许可指南、日本国籍法相关条文、日本所得税/法人税/消费税/居民税规则等具体条款/章节。
- 财税架构需更具体：日本非永住者/永住者税务居民规则、5年/10年、国外所得汇入、法人税/消费税/源泉税、役员报酬、经营管理公司实质、CFC/CRS、中国税务居民关系。
- 材料清单需拆开 HSP 和经营管理两条路径，不能混成一张概括表。

整改要求：单独升级日本 V21 终审版，补真实费用报价、入管/税法条款、HSP 积分证据链、经营管理办公室/资本金/商业计划/税务材料。

### 7. 新西兰 · SMC / AIP 技术与投资路径

当前优点：已补充 SMC 6分、55岁以下、英语、健康品行、认证雇主 skilled employment/job offer；AIP NZD 500万 Growth/3年、NZD 1000万 Balanced/5年方向正确。

不达标点：

- SMC 与 AIP 是两类完全不同项目，费用、材料、三方费用、三方材料、法案条款需要分开，不应只作为“SMC/AIP 技术与投资路径”混合表达。
- 财税架构需说明新西兰 IRD 税务居民、过渡居民 transitional resident、境外信托、投资收益、PIE、FIF、双重税收协定、CRS、中国税务居民和资金出境的影响。
- 需要逐条列 Immigration Act / Immigration Instructions 对 SMC、AEWV/认证雇主、AIP acceptable investments、residence conditions 的具体章节或操作指引。
- AIP 投资三方费用需明确基金/券商/托管/投资顾问/律师/税务师/审计报告/年度投资报告费用和材料。
- 佣金页不能只引用源页，需要在总佣金页展示真实结构化数据。

整改要求：拆成 SMC 与 AIP 两个最小单元分别达标，再融合；补 IRD 税务居民与投资税务专项。

### 8. 美国 · EB-1A 杰出人才绿卡

当前状态：明确为 `V20Plus定稿源 / V21融合包装`。**前置不合格。**
