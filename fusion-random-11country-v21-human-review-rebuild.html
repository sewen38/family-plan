旧版失败原因与本版修正动作

V21 11国随机组合最终审核页差距审计报告

审计对象：family-plan-pages/fusion-random-11country-v21-final-review.html

审计标准：family-plan-full Skill 内 V21 最终商业级执行策划案标准、v21-final-exec-standard.md、多国多项目融合标准、V21 人工审核新增硬标准。

审计结论：未达 V21 最终人工审核标准，不能作为最终版交付。

一、总体判定

页面在形式层面已经具备：Hero、数据完整性检查表、完整单项目模块嵌入区、15章归并正文、费用/材料/三方费用/三方材料关键词、财税章节、法案章节、佣金页等结构。

但按 V21 最终人工审核硬标准，核心问题不是“有没有章节”，而是：

1. 部分项目仍明确使用 V20Plus fallback，并套 V21 融合包装，不满足“每个选中项目必须先有合格 V21-professional 单项目完整页”的前置条件。

2. 真实数据不足：大量核心费用、政府费、服务费、底价、佣金、基金发行费、管理费、尽调费、三方费仍写“待确认 / 以官方规则为准 / 按官方费用表逐项核算 / 版本日期需复核”。这可用于内部草稿，不能用于 V21 最终交付。

3. 财税架构对税务问题的解决不够具体：部分国家只给“中国主体—境外SPV—区域公司—信托/基金会—家庭账户”等通用模板，未按国家/项目说明如何降低税负、避免双重征税、降低预提税、处理 CFC/CRS/FATCA/FBAR、利润归集、税务居民切换、37号文/ODI/FDI 与资金回流。

4. 法案条款颗粒度不均衡：澳大利亚 482、部分土耳其内容较接近要求；但日本、美国、多米尼克、格鲁吉亚、马耳他等多处仍是“官方移民局/内政部、税局/财政部、项目法案/条例/政策手册”这种类别级占位，没有逐条列出正式法案名称、条款/章节/政策段落、适用条件、风险、执行动作。

5. 佣金页不合格：多国总佣金页写“详见对应单项目源页佣金/成本区”“通常3个月或按项目进度”“根据实际情况”；多个单项目底价、佣金、对接人、结佣批次为待确认。V21 要求佣金页结构化展示选中项目的服务费、底价、佣金、结佣周期、结佣批次、对接人、状态，不得用泛化引用替代。

6. 所谓“完整单项目模块嵌入区”更多是源页链接/展开入口，不等于所有项目已通过 V21 单项目验证。尤其 fallback 项目不能因被嵌入/链接就视为 V21 合格。

本页 HTML 规模约 823KB，提取正文约 17.4万字、表格 499 个、SVG 28 个、图片 28 个。数量指标看似足够，但 V21 新硬标准明确：结构、表格、SVG 达标不代表通过；缺真实数据、缺有效财税避税架构、法案条款不详细，一律未达标。

二、失败原因分类

A. 前置单项目质量失败：fallback 冒充 V21 最终源

页面自身“数据完整性检查表”明确显示以下项目为 V20Plus定稿源 / V21融合包装：

  • 美国 · EB-1A 杰出人才绿卡：final-single/us-eb1a-v20plus-final.html
  • 多米尼克 · CBI 捐款入籍：final-single/dominica-cbi-v20plus-final.html
  • 格鲁吉亚 · 投资居留路径：final-single/georgia-investment-residence-v20plus-final.html
  • 马耳他 · MPRP 永居路径：final-single/malta-mprp-v20plus-final.html

V21 规则明确：fallback 只能作为旧参考，必须先升级为合格 V21-professional 单项目页，再参与最终商业级融合。当前页面把 fallback 项目放进 11 国融合，是根本失败项。

B. 数据真实性失败：核心数据大量待确认或泛化

典型问题：

  • 土耳其基金入籍费用表中：基金认购/发行费、基金管理费/托管费、政府申请费、基金尽调费、医疗保险/居留配套、服务费、E-2 二阶段评估费等多项为“待确认”。
  • 日本费用表中：行政书士、翻译公证、服务费/底价/佣金等仍待确认或区间过宽。
  • 香港、美国、多米尼克、格鲁吉亚、马耳他佣金/成本均存在大面积待确认。
  • 多国总佣金页没有真实底价、真实佣金、真实结佣批次、真实对接人，只写“详见对应单项目源页佣金/成本区”。
  • 部分费用写“按官方费用表逐项核算”“递交前复核版本”，属于流程提醒，不是已核验数据。

V21 要求费用、投资额、官方费、律师/顾问费、生活费、税率、门槛、时间、预算、结佣信息必须真实且详细；确实未知时要写未知原因、核验来源、影响范围，但不能用未知项替代核心交付数据。当前不满足。

C. 财税架构有效性失败：通用架构多，项目化税务解决不足

页面反复出现类似表述：

  • “中国主体—境外SPV—区域公司—信托/基金会—家庭账户”
  • “嵌入完整财税执行策划案框架：ODI、37号文、红筹、SPV、CRS、CFC、转让定价”
  • “按客户企业和家庭信息替换”
  • “用于本项目落地时按某项目替换参数”

这些表达说明财税内容仍是通用框架/参数替换思路，而不是针对每个国家/项目的税务问题解决方案。V21 要求第6章/第14章必须解释：

  • 税务居民如何判定与切换；
  • 境内企业利润、分红、股权转让、服务费、知识产权费、投资收益如何归集；
  • 哪个主体持有资产，哪条资金流合规出境；
  • 如何避免双重征税、降低预提税、处理 CFC/CRS/FATCA/FBAR；
  • 何时需要 37号文、ODI、FDI、返程投资、转让定价文档；
  • 对客户企业和家庭税负有什么实际优化结果。

当前很多国家没有把上述内容落到项目和法域，尤其 fallback 项目基本是财税模板套壳，未能证明“解决税务问题”。

D. 法案条款失败：法规附件存在类别级占位

合格示例：澳大利亚 482 有较具体条款，如 Migration Act 1958 s140E、s245AR/s245AS，Migration Regulations 1994 reg 2.59、reg 2.72，Subclass 482、Subclass 186、Fair Work Act 等，并说明客户动作和风险。

不合格示例：日本、美国、多米尼克、格鲁吉亚、马耳他等章节出现统一模板：

  • 官方移民局/内政部：项目资格、表格、审理流程、费用表；
  • 税局/财政部：税务居民、个人税、公司税、资本利得、申报期限；
  • 项目法案/条例/政策手册：投资门槛、持有期、续签/入籍、尽调标准；
  • 学校/医保/福利官方资料：教育、医疗、福利资格。

这不是 V21 所要求的“法案/政策名称 + 条款/章节/政策段落 + 适用项目 + 适用条件 + 约束/风险 + 执行动作”。类别级来源不能通过人工审核

第1章 客户家庭基本信息

本次为11国随机组合压力测试，客户画像采用“高净值中国企业主家庭 + 多国身份/经营/教育/资产配置需求”的默认审核画像；正式客户版必须替换为真实客户家庭成员、税务居民、企业、资产、资金来源、教育与居住计划。

第2章 核心策略：按税务功能分层，而不是堆护照

功能层国家/项目核心作用税务边界
低税经营与利润归集新加坡EP/PIC、香港ASMTP、格鲁吉亚投资居留真实区域总部、贸易/控股/远程服务节点必须有人员、合同、董事决策、银行、风险承担；不能空壳截留中国利润

第3章 合规清理详细方案

  • 资金来源：每笔投资款必须形成“收入/分红/资产出售/贷款/赠与/继承 → 纳税/完税 → 银行流水 → 出境路径 → 项目收款方”的闭环。
  • 企业KYB：新加坡、香港、格鲁吉亚、日本经营管理、澳洲雇主担保必须审查真实业务、办公室、员工、合同、税务与银行流水。
  • 中国侧：37号文、ODI、FDI、个人购汇、境外存量资金不得混用；不得通过地下钱庄或虚假贸易转移资金。
  • 国际侧：CRS/FATCA/FBAR自证必须与真实税务居民一致。

第4章 境外资金归集与投资架构

资金路径按三类归集：经营利润路径（新加坡/香港/格鲁吉亚）、身份投资路径（土耳其/多米尼克/马耳他/新西兰AIP）、迁入前资产保护路径（美国/澳洲/日本/新西兰）。每个项目的具体金额、收款方、付款节点见后方“费用明细/三方费用”。

第5章 投资使用建议

非投资项目（EP、ASMTP、482、HSP、SMC、EB-1A）不得把生活费或服务费伪装成投资门槛；投资/捐款项目（土耳其基金/房产、多米尼克、格鲁吉亚、马耳他、新西兰AIP）必须分开列示可退出投资、不可退政府费、专业服务费和生活安顿费。

第6章 财富架构搭建方案

本版已逐项目重画“解决税务问题”的架构：每张图必须解释税务居民、资金流、利润归集、DTA/预提税、CFC/CRS/FATCA/FBAR、37号文/ODI/FDI，而不是泛泛身份图。详见后方每个项目的“税务架构图 + 税务架构重设计全文”。

第7章 税务分析

低税经营节点强调真实商业实质；高税迁入节点强调登陆/长期居住前完成资产重组；护照/居留备份节点强调不自动改变纳税地。第14章保留组合式财税总图和逐项目税务全文。

第8章 资金跨境合规方案

  • 个人路径：年度便利化购汇、境外存量资金、已完税分红/资产出售款，必须保留来源文件。
  • 企业路径：ODI/对外投资、服务贸易、真实采购/销售、转让定价资料。
  • 返程路径：境外SPV回投中国需FDI与37号文评估。
  • 严禁：虚假服务费、代持隐匿、地下钱庄、护照冒充税务居民。

第9章 身份路径规划

本组合覆盖工作/经营/投资/永居/护照五类路径。每个项目的官方门槛与适用条件见后方“来源补强全文”和“法案条款与官方来源”。

第10章 教育规划

教育规划按居住国分层：新加坡/香港/澳洲/日本/新西兰/美国为实际居住与教育主场；马耳他提供欧盟居留备选；土耳其/多米尼克/格鲁吉亚更多是身份或经营备选，不应承诺核心教育福利。

第11章 福利居住国规划

# 10. 格鲁吉亚投资居留：低税经营/个人税务居民备选与欧亚业务节点

## 客户税务痛点

- 客户希望利用格鲁吉亚相对友好的税制和投资居留作为欧亚业务、轻资产经营或个人税务居民备选。
- 需要判断是否真的能改变中国/高税国家税务居民，而不是只取得居留。
- 跨境电商、咨询、IT、贸易收入如果由格鲁吉亚公司承接，需证明实质和客户来源。

## 架构节点

- **格鲁吉亚公司/个体经营节点**：承接欧亚/远程服务、贸易、IT、咨询等真实业务。
- **中国经营主体**：保留中国团队和中国来源利润。
- **香港/新加坡控股或收款节点（可选）**：如客户/银行需求更适合，可与格鲁吉亚分工。
- **个人税务居民节点**：是否满足格鲁吉亚税务居民条件及是否脱离原居民国。
- **家庭信托/保险/投资账户**：上层传承与资产配置。

## 资金流

1. 境外客户与格鲁吉亚公司签约、付款。
2. 格鲁吉亚公司支付本地工资、办公、外包和税费。
3. 中国公司如提供服务，应按合同收取服务费并在中国纳税。
4. 格鲁吉亚公司完税后分红至个人/控股公司/信托。

## 利润归集

- 可归集格鲁吉亚公司实际承担的销售、交付、技术、管理利润。
- 如核心团队仍在中国，仅由格鲁吉亚公司开票，会被视为利润错配。
- 可设计为：格鲁吉亚承担客户签约与交付，中国公司承担外包研发/支持并收成本加成。

## 预提税 / DTA

- 格鲁吉亚与部分国家有 DTA，但网络和适用效果需逐项核查。
- 向中国支付服务费、股息、利息、特许权使用费需看来源地和 DTA。
- 分红、利息、特许权使用费可能有格鲁吉亚本地预提税。

## CFC / CRS / FATCA / FBAR / 37号文 / ODI / FDI / 当地税务居民影响

- **CFC**：中国或高税国居民控制格鲁吉亚低税公司，需合理商业目的和实质。
- **CRS**：格鲁吉亚金融账户合规程度需按银行要求填写税务居民信息；不能隐瞒。
- **FATCA/FBAR**：美国身份客户仍需披露。
- **37号文/ODI/FDI**：中国个人/公司出资格鲁吉亚公司需评估外汇及 ODI。
- **当地税务居民**：成为格鲁吉亚税务居民需满足居住或高净值等条件，同时还需处理原居民国退出/双重居民冲突。

## 为什么能合法降税或隔离风险

- 若客户确有欧亚、远程服务、IT、贸易业务，把职能、合同、银行、雇员、管理放在格鲁吉亚，可以形成低成本经营节点。
- 与中国主体通过转让定价分工，减少中国利润被无依据外移的风险。
- 格鲁吉亚居留作为备选迁移地，有助于高税国家迁入前的税务居民路径设计。

## 不可做事项

- 不可只注册格鲁吉亚公司而无办公、人员、客户管理即截留利润。
- 不可用格鲁吉亚账户收中国境内业务收入。
- 不可宣称取得投资居留就自动免中国税。
- 不可忽略原税务居民国 CFC 和 CRS。

## 可转 SVG 结构说明

境外客户 -> 格鲁吉亚公司[合同/交付/收款/本地实质]

格鲁吉亚公司 -> 外包/采购 -> 中国经营主体[成本加成/中国纳税]

格鲁吉亚公司 -> 分红 -> 个人/控股公司/信托

个人 -> 格鲁吉亚税务居民评估 + 原居民国退出评估

# 11. 马耳他 MPRP：欧盟居留、非定居税制机会与高合规资产持有节点

## 客户税务痛点

- 客户希望通过马耳他 MPRP 获得欧盟居留和家庭备份，但不清楚居留、税务居民、domicile、remittance basis 的区别。
- 如果实际迁居马耳他，海外收入/资本利得、汇入款、公司分红如何纳税需要提前规划。
- 欧盟银行、CRS、反洗钱、资金来源审查严格，不能用 MPRP 隐匿资产。

## 架构节点

- **原税务居民国资产池**：中国/香港/新加坡/其他国家资产。
- **马耳他居留节点**：MPRP 居留权，不等于自动税务居民。
- **马耳他个人税务居民但非 domicile（如适用）**：可能适用汇入制逻辑，需专业确认。
- **香港/新加坡/离岸控股公司**：承接真实非马耳他业务。
- **家庭信托/保险**：需考虑马耳他、原居民国及未来继承地税务。
- **欧盟投资账户/银行账户**：高 KYC/CRS 合规。

## 资金流

1. 已完税资金支付 MPRP 捐款、租房/购房、行政费。
2. 若不成为马耳他税务居民，全球所得仍按原居民国申报。
3. 若成为马耳他税务居民，海外收入汇入马耳他、马耳他来源收入、资本利得按当地规则处理。
4. 海外公司分红/投资收益是否汇入马耳他，需年度现金流规划。

## 利润归集

- 马耳他 MPRP 本身不是经营利润归集工具。
- 非马耳他业务利润应归集在香港/新加坡/中国/格鲁吉亚等真实经营地。
- 若设马耳他公司，必须有欧盟业务、董事、办公、会计和实质，否则税务成本与合规成本不匹配。

## 预提税 / DTA

- 马耳他 DTA 网络较广，但 MPRP 个人是否能享受、如何享受要看税务居民证明和受益所有人。
- 海外公司向马耳他居民分红，可能在来源国预提，在马耳他按汇入/来源规则处理。
- 欧盟内支付还需考虑反避税指令、受益所有人、经济实质。

## CFC / CRS / FATCA / FBAR / 37号文 / ODI / FDI / 当地税务居民影响

- **CFC**：若客户仍为中国/澳洲/美国等居民，马耳他居留不改变 CFC 适用。
- **CRS**：马耳他和欧盟金融机构严格 CRS 交换。
- **FATCA/FBAR**：美国身份客户持有马耳他/欧盟账户必须申报。
- **37号文/ODI/FDI**：中国侧境外持股和资金出境仍需合规。
- **当地税务居民**：MPRP 居留 ≠ 马耳他税务居民；实际居住、住所、生活中心、税务登记决定纳税范围。

## 为什么能合法降税或隔离风险

- 其税务价值在于**欧盟居留备份 + 可规划的税务居民/非 domicile 路径 + 高合规资产持有环境**。
- 对有欧盟生活、子女教育、医疗需求的家庭，可在不立即改变全部税务居民的情况下建立备选通道。
- 若未来迁居，可通过汇入规划、海外公司实质、DTA 抵免降低重复征税。

## 不可做事项

- 不可把 MPRP 居留当作自动税务居民证明使用。
- 不可用马耳他账户隐藏中国/美国/澳洲税务居民身份。
- 不可无实质设马耳他公司套用 DTA。
- 不可把海外收入随意汇入马耳他后再主张免税。

## 可转 SVG 结构说明

已完税家庭资金 -> MPRP费用/租购房 -> 马耳他居留节点

原税务居民国 -> 继续全球/来源地纳税判断

香港/新加坡/中国/格鲁吉亚公司 -> 真实业务利润归集

欧盟银行/投资账户 -> CRS/FATCA/KYC披露

# 12. 11国组合下的推荐分层架构：按税务功能而非护照数量搭建

## A. 低税经营与利润归集层

- 首选：**新加坡 EP/PIC** 用作东盟/全球区域总部、IP/贸易/投融资节点。
- 辅助：**香港 ASMTP** 用作大湾区贸易、融资、控股、收款节点。
- 机会型：**格鲁吉亚投资居留** 用作欧亚远程服务、IT、轻资产经营节点。

关键规则：三者不能重复承担同一利润功能。每个节点必须有人员、合同、银行、董事决策、客户、成本和风险。

## B. 高税迁入保护层

- **美国 EB-1A**：最高优先级，登陆前必须完成 PFIC/CFC/信托/遗产税规划。
- **澳大利亚 482**：迁入前做资产估值、信托/保险、境外公司管理地隔离。
- **日本 HSP/经营管理**：关注非永久居民窗口、海外资产申报、继承赠与税。
- **新西兰 SMC/AIP**：利用过渡居民窗口和 FIF/CFC 规则进行投资重组。

关键规则：高税国家身份不是税务优化工具，而是风险触发器。必须先规划，再登陆/长期居住。

## C. 投资与身份备份层

- **土耳其基金/房产**：身份投资、不动产/基金资产分散，但不改变税务居民。
- **多米尼克 CBI**：护照与通行备份，不是税务居民切换。
- **马耳他 MPRP**：欧盟居留备份，可规划税务居民/非 domicile 路径，但需实居和汇入规则判断。

关键规则：身份文件、居留卡、护照不是纳税地。纳税地由居住、住所、生活中心、国籍/绿卡、实际管理地和 DTA 决定。
